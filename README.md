# StarsHub

> uTools GitHub 星标管理插件 -- 以维度标签、视图、语义搜索为核心的 Stars 管理体验

## 设计哲学

### 1. 维度标签替代默认分类 -- 把分类权交给用户

默认分类常分错且维度单一固定。StarsHub 让用户自定义"维度"（如领域、类型、技术栈），每个维度有多个带描述的选项。AI 分析时在用户定义的选项里约束式选择，而不是自由生成标签。标签结构化、可枚举、打错能手动修正。

### 2. 视图 = 标签组合 -- 浏览即过滤

视图是用户定义的标签组合过滤器（如"领域=AI AND 语言=Python"）。点侧边栏的视图，内容区立即切到对应仓库子集。视图定义纯标签，语义搜索不进视图定义，避免"两道 AI 叠加导致准确性下降"。

### 3. 语义搜索本地化 -- 摆脱外部服务依赖

利用 uTools preload.js 的 Node.js 环境（无 CORS 限制），直接调 embedding API，向量存 utools.db 分片，搜索时 JS 端 brute-force cosine 相似度。零外部服务、零部署成本。

### 4. 关键词 + 语义协作

uTools 子输入框做关键词实时搜索（快速直达），工具栏搜索框做精确筛选，⚡ 按钮切换语义增强。两者结果融合，用户一次看到全部相关仓库。

### 5. 趋势联动关注重点

不是被动逛榜单，而是主动发现。用 GitHub Search API 拉取近期热门仓库，AI 判断哪些匹配用户预设的关注重点，匹配的特殊标注。

### 6. 双 AI 后端

分析类（打标签/特色分析/趋势匹配）默认走 utools.ai()，设置里可切换自定义 API。embedding 永远走自定义（utools.ai() 没有 embedding 接口）。

### 7. 侧边栏布局

窄高型窗口左右压缩能接受，上下压缩很难受。左侧边栏三段式固定布局：视图区在顶部，操作区+功能区固定在底部。

## 技术栈

React 19 + TypeScript + Vite 6 + Tailwind CSS 4 + Zustand 5

## 开发

```bash
npm install
npm run build
```

在 uTools 开发者工具中导入项目根目录即可。
